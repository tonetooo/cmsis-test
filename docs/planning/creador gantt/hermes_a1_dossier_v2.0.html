Resumen Ejecutivo

HERMES-A1 (AWTAS — Autonomous Wireless Triaxial Acquisition System) es un sistema de adquisición de datos triaxiales autónomo, diseñado para operar en entornos industriales y remotos sin intervención humana.

¿Qué es HERMES-A1?

HERMES-A1 es un instrumento de medición de tercera generación que integra un acelerómetro digital de alto rendimiento ADXL355, un microcontrolador STM32F446RE con FreeRTOS, almacenamiento local en SD, y conectividad celular 4G LTE mediante módulo Quectel EC25.

Los datos adquiridos son transmitidos de forma segura a Google Drive a través de un backend Flask con dashboard web, permitiendo la configuración remota y la visualización en tiempo real del estado del equipo sin necesidad de infraestructura de red local.

🔧 Diferenciadores Clave

  • Conectividad: 4G LTE nativa (no requiere WiFi ni Ethernet)
  • Autonomía: Operación continua sin intervención humana
  • Configuración remota: Vía cloud (parámetros en vivo)
  • Wake-on-Motion: Detección por umbral configurable, bajo consumo
  • Almacenamiento: SD local + RAM buffer + Cloud (redundante)
  • Backend: Flask API + Dashboard web propio

El Problema y la Solución

En entornos industriales y remotos, el monitoreo de vibraciones enfrenta barreras significativas que HERMES-A1 resuelve de manera integral.

El Desafío

  • Ubicaciones sin conectividad (minería, torres, estaciones remotas)
  • Costos operativos elevados (desplazamiento de personal)
  • Falta de tiempo real (datos días/semanas después de captura)
  • Condiciones adversas (temperaturas extremas, humedad, vibración constante)

La Solución HERMES-A1

  • 4G LTE nativa: Sin necesidad de WiFi o Ethernet
  • Autonomía total: Operación continua sin intervención humana
  • Configuración remota: Vía cloud con parámetros en vivo
  • Wake-on-Motion: Bajo consumo, detección inteligente de eventos
  • Redundancia: SD local + RAM buffer + Cloud
  • Dashboard web: Gestión completa desde cualquier lugar

3.2 Formato de Datos

Formato CSV:

timestamp_rel_s;timestamp_abs;unix_time;x_g;y_g;z_g;voltaje;corriente;potencia

Descripción:

  • timestamp_rel_s: Tiempo relativo desde inicio de captura (s)
  • timestamp_abs: Marca de tiempo absoluta (Unix epoch)
  • unix_time: Tiempo Unix (redundancia)
  • x_g, y_g, z_g: Aceleración en cada eje (g)
  • voltaje: Tensión de alimentación (V)
  • corriente: Corriente del sistema (A)
  • potencia: Potencia calculada (W)

Delimitador: punto y coma (`;`) — compatible con locales de coma decimal.

5.2 Wake-on-Motion — Detección Inteligente de Eventos

El sistema permanece en estado de bajo consumo hasta que detecta actividad sísmica/vibratoria que supera el umbral configurable (TRIGGER_G). Este mecanismo permite:

  • Capturar solo lo relevante: Sin gigabytes de datos en reposo.
  • Autonomía extendida: Ideal para monitoreo en batería o energía limitada.
  • Configuración remota: Umbral ajustable sin intervención en terreno.

Aplicaciones y Casos de Uso

HERMES-A1 es una solución versátil para múltiples aplicaciones de monitoreo estructural.

8.1 Monitoreo de Vibraciones Industriales

  • Maquinaria rotativa: Motores, bombas, ventiladores, compresores, turbinas.
  • Detección temprana de desgaste: Rodamientos, engranajes, ejes.
  • Análisis de firma espectral: Identificación de frecuencias características de falla.
  • Mantenimiento predictivo: Alarmas automáticas ante niveles anormales.

8.2 Monitoreo Estructural (SHM)

  • Puentes y viaductos: Respuesta dinámica ante carga de tráfico y viento.
  • Edificios y torres: Monitoreo de vibraciones ambientales y sísmicas.
  • Presas y taludes: Detección de movimientos precursores de inestabilidad.
  • Túneles y minería subterránea: Monitoreo de vibraciones por voladuras.
